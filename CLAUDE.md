# ArgentVigil

Silver speculative-positioning monitor with gold as comparative context. Framing is **"selling dollars, not buying metals"** — is the speculative futures crowd genuinely capitulated, or just pulling back. Not a trading system: no price targets, no prediction framing, no risk-tolerance commentary. See `SPEC.MD` (gitignored, local-only — do not assume it exists in fresh clones) for the full Stock & Flow panel spec, and `README.md` for the user-facing feature/data-source overview.

## Architecture

Two independent layers that share one frontend:

1. **CoT pipeline** (`pipeline/`) — stdlib-only Python, no server. Fetches CFTC positioning data + Yahoo Finance prices, computes signals, writes a static JSON cache. Run manually/on a schedule, not on every request.
2. **Exchange inventory backend** (`main.py`, `db.py`, `mc_token.py`) — FastAPI server, proxies metalcharts.org live, persists to SQLite, serves the built frontend as static files.

```
main.py            FastAPI app: proxy routes for COMEX/SHFE/PSLV/prices, market-balance endpoint, serves frontend/dist
db.py               SQLite persistence (sqlite3 stdlib, no ORM) — argentvigil.db at repo root
mc_token.py         metalcharts.org auth token fetch/cache (module-level globals, single-process only)
silver_market_balance.json   Manually maintained annual Silver Institute balance data (Layer 3 of SPEC.MD)

pipeline/
  config.py         All tunable thresholds/constants — edit here, not inline
  fetch.py          CFTC Socrata API (urllib, stdlib only)
  price_fetch.py     Yahoo Finance chart API (urllib, stdlib only)
  compute.py        net_long_pct_oi, percentile ranks, signal classification, track record
  run.py             Entry point — orchestrates fetch -> compute -> cache/cot_data.json
  cache/cot_data.json   Pipeline output (gitignored, regenerated by run.py)

frontend/
  src/App.jsx                Top-level composition: CoT tracker + Inventory dashboard + Macro watchlist
  src/silver_cot_tracker.jsx CoT panel (chart, signal banners, track record, MacroWatchlist component)
  src/comex_inventory.jsx    Exchange Inventory Dashboard (COMEX/SHFE/PSLV panels, largest file)
  src/market_balance.jsx     Stock & Flow Layer 3 UI (annual balance chart) — built but NOT yet imported into App.jsx
  src/palette.js             Shared chart color constants (VAULT_COLORS)
  vite.config.js             publicDir points at pipeline/cache (serves cot_data.json in dev); /api proxied to :8000

utils/
  dev.sh              Boots venv, installs deps if missing, runs FastAPI (:8000) + Vite (:5173) together
  sniff-metal-charts.py, sniffer.sh   Tools for reverse-engineering metalcharts.org API responses
```

**Convention:** runnable scripts live in `utils/`, not the repo root (established preference — keep new launchers/helpers there).

## Data flow

- **CoT data**: `pipeline/run.py` → `pipeline/cache/cot_data.json` → read directly by the frontend via Vite's `publicDir` (dev) / served from `frontend/dist` (prod, since `vite build` copies `publicDir` contents in). No backend involvement.
- **Exchange inventory**: frontend calls `/api/*` → FastAPI proxies metalcharts.org (authenticated via `mc_token.py`) → response upserted into SQLite → returned to frontend. `/api/*/db/*` endpoints read straight from SQLite instead of hitting metalcharts.org (cheap history reads).
- **Market balance**: `main.py` reads `silver_market_balance.json` directly (no DB, no external fetch) and computes derived metrics (5yr cumulative, runway years, staleness) on each request.
- **Macro watchlist**: manually edited fields inside `cot_data.json` (`macro_watchlist` key). `run.py` preserves this block across runs — never overwrite it programmatically.

## Conventions

- **Backend**: stdlib `sqlite3` (no ORM), `httpx.AsyncClient` for outbound calls (one shared client via FastAPI lifespan), routes are thin — fetch, reshape, upsert, return `{"success": bool, "data": ...}`. HTTP errors from upstream become `HTTPException(502, str(e))`.
- **Pipeline**: intentionally stdlib-only, no third-party deps — keep it that way (it's meant to run standalone without the venv). Config constants belong in `pipeline/config.py`, not scattered inline.
- **Frontend**: React 19 + Vite 5 + Recharts, no state library, no router (single page, tab-like sections via `App.jsx`). Chart colors come from `palette.js`, not ad hoc hex values.
- **Units**: COMEX figures are troy oz; SHFE is kg natively, converted via `1 kg = 32.1507 oz` (constant appears in both `main.py` and any frontend display — keep consistent). COMEX open interest contracts are 5,000 oz each.
- **Nulls over zeros**: metalcharts.org sometimes reports `0` to mean "not reported that day" — `_parse_aggregate_row` in `main.py` converts those to `None` so charts gap instead of showing false dips. Apply the same treatment to any new fields with this failure mode.
- **AV Voice Rules** (from SPEC.MD, applies repo-wide to any user-facing copy): no "alarming"/"critical" framing on deficit trends, always show the ±20% uncertainty qualifier on above-ground stock figures, label runway/track-record figures explicitly as historical description or context — never as prediction.

## Running it

```bash
bash utils/dev.sh          # everything (venv bootstrap + backend :8000 + frontend :5173)
python3 pipeline/run.py    # CoT pipeline only, no server needed
```

Run the pipeline at least once before the frontend, since the CoT tab reads directly from `pipeline/cache/cot_data.json`.

## Data sources (see README.md for full table)

CFTC PRE Socrata API (CoT), Yahoo Finance (SLV/GLD/GC=F/SI=F), metalcharts.org (COMEX/SHFE inventory, volume/OI, delivery notices, spot prices — reverse-engineered API, auth via `mc_token.py`), Sprott direct API (PSLV), Silver Institute World Silver Survey (manually transcribed, annual).
